@Test
каждый тест метода помечается аннотацией тест. Для работы каждого теста создается отдельный объект класса test
(обеспечение не вмешательства тестов друг для друга).
Доказательство в проекте my_junit NewInstanceForEachTest. В первом тесте поле изменяется с помощью сеттера
во втором тесте поле принимает исходное состояние.

Переменная статик не сбрасывается т.к. принадлежит классу

-----------------------------------------------------------------------------------------------------------------------
@BeforeEach
Метод помеченный аннотацией @BeforeEach называется init() или setUp() выполняется каждый раз перед выполнением каждого
метода помеченного аннотацией тест
-----------------------------------------------------------------------------------------------------------------------
@AfterEach
Метод помеченный аннотацией @AfterEach называется destroy() или tearDown() выполняется каждый раз после выполнения каждого
метода помеченного аннотацией тест. Аннотация @AfterEach была создана для возврата ресурсов в систему. Если
вдруг понадобится открыть соединение и т.п.

-----------------------------------------------------------------------------------------------------------------------
@BeforeAll
Метод помеченный аннотацией @BeforeAll вызывается один раз при инициализации класса. Метод должен быть статическим

-----------------------------------------------------------------------------------------------------------------------
@AfterAll
Метод помеченный аннотацией @AfterAll вызывается один раз при выгрузке класса. Метод должен быть статическим


----------------------------------------------------------------------------------------------------------------------
@DisplayName("текст, выводимый на экран при прогонке теста вместо названия метода")
Аннотация заменяющая название метода в списке прогоняемых тестов
