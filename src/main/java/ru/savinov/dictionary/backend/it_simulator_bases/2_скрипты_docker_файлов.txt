файл для скриптов автоматизации по умолчанию называется Dockerfile без расширения.
Инструкции Dockerfile можно условно разбить на две группы:

 - Те, которые запускаются при сборке образа
 - Те, которые запускаются при старте контейнера

FROM - первая инструкция в скрипте, которая указывает, на основе какого базового образа будет собран текущий образ

RUN - здесь указываются команды (чаще всего операционной системы), которые будут выполнятся при сборке образа

LABEL - позволяет указать порт(ы) в вашем контейнере, которые будут доступны извне, скорее всего для других контейнеров

ENV - здесь вы задаете переменные окружения в образе и если нужно указываете их начальные значения

ADD - копирует файлы или папки из вашей файловой системы в образ. Также поддерживает работу с архивами и удаленными файлами (Http)

COPY - копирует файлы или папки из вашей файловой системы в образ

ENTRYPOINT - указывает, какая команда будет запущена при старте контейнера

CMD - так же указывает, какая команда будет запущена при старте контейнера. Здесь же можно указать параметры для
ENTRYPOINT. Имеет несколько форм записи, как и предыдущая инструкция.

WORKDIR - указывает текущую папку в образе/контейнере при работе инструкций
