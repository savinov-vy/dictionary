Топик
Топик это набор не структурировананных сообщений (некоторый аналог таблицы в БД)
для кафки сообщение это просто набор байт, по верх которых можно использовать разные протоколы
- JSON, Protobuf, Avro и т.д.

Слишком большое количество (больше 1000) топиков (и партиций) в одном брокере приводит к снижению производительности.
Поэтому слишком дробить на топики не стоит.

топик состоит из партиций
================================================
         /-partition-1: 0 1 2 3 4 5 6 7 8

 topic   --partition-2: 0 1 2 3 4

         \_Partition-3: 0 1 2 3 4 5 6

            Old---------->>>>>>>New
 producer может писать сообщения в разные партиции
 партиции физически находятся на разных машинах и обрабатываются разными consumer из одной группы
 таким образом распределяется нагрузка между нодами
