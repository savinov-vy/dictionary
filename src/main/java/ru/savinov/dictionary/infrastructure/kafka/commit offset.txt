commit offset
это тот офсет который был сохранён в kafka
кафка работает в batch режиме. т.е. передаёт сообщения консюмеру пачками, таким образом увеличивается
быстродействие. Так вот если консюмер взял пачку сообщений, обработал только часть из них и упал
то при перезапуске он снова возьмет ту же пачку. Т.к. не передал commit offset на сторону kafka
с большой вероятностью консюмер снова упадёт и снова возьмет ту же пачку сообщений.

таким образом commit offset это тот offset сообщений, который был отработан.

commit offset хранится в kafka в специальном топике, где хранятся offset-ы для всех consumers для всех партиций
__consumer_offsets
