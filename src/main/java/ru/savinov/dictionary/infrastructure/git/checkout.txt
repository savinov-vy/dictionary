Команда служит для перемещения между ветками и коммитами

при перемещении между не актуальными коммитами указатель head находится
в состоянии deatached и поэтому коммиты из этого состояния не актуальны


-----------------
перейти к необходимомоу коммиту

git checkout COMMIT_ID


-----------------
перейти к актуальному коммиту

git checkout master

-----------------
на один коммит назад

git checkout HEAD^

-----------------
вернуть какой либо файл в состояние из конкретного коммита
вернуть файл какой-то версии в прошлом

git checkout commit_id -- file1 file2

git checkout commit_id -- путь_до_файла_1 путь_до_файла_2

вернёт файлы к версии указанной в коммитах

----------------
вернуть какой либо файл в состояние из последнего коммита
изменения должны быть не отслеживаемые

git checkout -- file1 file2

git checkout -- путь_до_файла_1 путь_до_файла_2

вернёт файлы к версии указанной в коммитах

git checkout -- . <- все неотслеживаеме изменения удалить

========================

удалить отслеживаемые изменения
git reset тоже самое что git reset --mixed HEAD <-перевести отслеживаемые изменения в неотслеживаемые
git checkout -- . 

другая версия удалить все отслеживаемые изменения
git reset --hard все отслеживамые изменения удалить к последнему коммиту


две черты означает что это не зарезервированное слово
git checkout master -> переключить на ветку мастер к последнему коммиту

git checkout -- master -> вернуть файл с названием master к версии последнего коммита
