опасная команда можно потерять данные!!!

Нельзя делать git reset к опубликованным коммитам на публичном репозитории!!!!
потому что если удалить последние коммиты, а ктото уже начал работу от последнего коммита
их работа так же будет потеряна (будет не ясно какая версия репозитория правильная)


с помощью команды можно вынести переместить указатель HEAD на любой коммит, а коммиты выполненные позже можно 
либо вынести в отслеживаемую зону, вынести в неотслеживаемую зону или удалить.

git reset --soft HEAD~2
или
git reset --soft HEAD^^
перевести указатель HEAD на 2 коммита назад
коммиты выполенные позже коммиты оставить в отслеживемой зоне

--------
git reset --mixed HEAD~3
или
git reset --mixed HEAD^^^
перевести указатель HEAD на 3 коммита назад
коммиты выполенные позже коммиты удалить, а изменения перевести в неотслеживаемую зону
--------

опасная команда!!!
git reset --hard HEAD~1
или
git reset --hard HEAD^
перевести указатель HEAD на 1 коммит назад
коммиты выполенные позже коммиты Удалить!!! 


---------

можно вернуть гит к указанному коммиту


git reset [--soft | --mixed | --hard] COMMIT_ID

# git reset --soft 4517ca72d380be4b30b585e57c5f2f8360403c3f


---------

по умолчанию используется --mixed
по умолчанию используется HEAD

git reset HEAD^^ = git reset --mixed HEAD^^

git reset = git reset --mixed HEAD
